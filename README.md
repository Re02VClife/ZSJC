

# 动画作画张数计数器

【项目简介】
本工具用于实时检测并统计动画视频中的“作画张数”（cel），
即每秒出现的新画面数量。通过屏幕截图连续分析相邻帧的变化，
检测流程融合了自适应二值化变化检测、快速全局相关系数、稀疏光流图层分离、
平均哈希重复帧过滤以及动态触发式完整过滤机制，
最终实现实时区分真正作画张数与运镜/局部变化，
自动排除摄像机运镜、图层平移、缩放等干扰，帮助你快速评估动画的作画密度和节奏。

新增 OP/ED 自动检测与扣除：利用图像哈希滑动窗口匹配，
自动识别并标记重复的片头片尾片段，从统计中扣除其张数与时长，
并提供运行记录保存功能，方便日后对比不同动画的作画数据。

这款轻量级工具能提供即时、直观的量化数据。
直观展示作画经费、便于进行各个动画之间的赛博斗蛐蛐。
仅限于2D动画、受算法局限性，检测出来的数据只是相对准确而不是绝对精准；
测出来高数据只代表原画、中割等制作人员付出的血汗较多并不意味着经费一定爆炸。

本项目纯练手，无任何参考，出现各种奇怪的BUG属于正常现象。
纯cpu计算，未使用CUDA，目前不知道兼不兼容Linux、mac。

## 功能特性

- **屏幕实时捕获**：基于 `dxcam` 高速截取屏幕，可自由调整截图区域。
- **双重检测管线**：
  - 基础检测：快速全局相关系数 + 差分，维持低负载运行。
  - 完整检测：全局平移对齐、光流图层分离、缩放检测、局部运动分析、背景相似度。
- **多种过滤策略**：
  - 全局平移运镜过滤（光流法）
  - 多层平移/图层分离过滤（光流方向一致性）
  - 缩放运镜过滤（光流径向一致性）
  - 重复帧过滤（感知哈希，汉明距离）
  - 局部变化过滤（连通域面积、包围盒、宽高比分析）
- **动态触发机制**：基础检测在短时间内检测到一定数量的新张后，
  自动开启完整过滤，平衡性能与准确度。
- **OP/ED 自动识别与扣除**：
  - 基于图像哈希（pHash）滑动窗口匹配，自动发现重复播放的 OP/ED 片段。
  - 状态机维护连续匹配，片段长度达标后记录并扣除其张数/时长，统计中只保留正片数据。
  - 支持精确定位修正（FFT 互相关），提高起止时间准确性。
- **运行记录管理**：可随时保存当前运行数据，
    支持查看、编辑、删除历史记录。
- **实时数据显示**：
  - 实时张数（张/秒）、总张数、运行时长、平均张数（扣除 OP/ED 后的净数据）
  - 智能状态提示（静止、运镜、一拍一/二/三等）
- **双波形图**：
  - 波形1（60秒）：实时过滤速率 vs 原始变化速率
  - 波形2（25分钟）：平滑后的长期趋势
  - 可配置的 OP/ED 背景高亮
- **可调参数面板**：所有检测阈值、开关、颜色均可在设置窗口实时修改并保存。
- **自定义截图区域**：提供可视化矩形拖拽界面，精确选定检测区域。
- **实时预览**：支持稠密光流、稀疏光流、差分残影、运动轨迹叠加等多种可视化。
- **布局管理**：支持多套窗口布局的保存与加载。
- **自动暂停**：指定秒数无新张后自动暂停，避免资源浪费。

## 安装

### 依赖
项目需要 **Python 3.11**(其他没试过)，并安装以下库：

```bash
pip install -r requirements.txt
```

### 项目文件结构
```
cel_counter/
├── main.py               # 单一合并文件（含全部逻辑）
├── requirements.txt      # 依赖列表
├── cel_counter_settings.json  # 运行时生成的配置
└── README.md
```

逻辑模块划分（均合并于 main.py）：
- config.py        # 全局配置、颜色管理器
- detection.py     # 差分、光流、哈希、缩放等过滤算法
- preview.py       # 实时预览渲染
- ui.py            # GUI 界面（tkinter）
- main.py          # 主程序入口与循环

## 快速上手

### 1. 运行程序
```bash
python main.py
```
窗口将以透明无边框形式出现在屏幕左上角。鼠标尖尖对准文字，长按即可实现拖动。

### 2. 设定截图区域
点击「设置」→「显示截图范围」，拖动红色矩形框选择要分析的画面区域，点击矩形外任意位置或在设置中关闭。

框的大小在设置面板中调整（截图区域比例）CROP_RATIO 自动计算。

### 3. 开始检测
播放动画或任意视频（最好是2D动画），程序运行时会自动统计新作画张数。
第一次使用本程序时，确保显示器的刷新率大于60hz，
在红框范围内做一些较猛的变化（如网页页面快速上下翻滚、拖动窗口猛甩、高帧率3D游戏转动视角等）
过程中看到左上角实时张数大于40（峰值48）即为设备性能合格，24以下测的不准。

### 4. 查看结果
- 左上角信息面板：实时张数、总张数、净平均张数（扣除 OP/ED）、当前状态。
- 右侧两个波形图：短期与长期张数速率变化，已识别的 OP/ED 区间会高亮显示。
- 点击设置，查看预览图和更详细的过滤信息、总张数历史及各类过滤数量。

### 5. 保存运行记录
点击设置中的「保存运行记录」按钮，输入名称即可保存当前运行数据，
包含 OP/ED 扣除明细、各类过滤数量、瞬时帧率统计等。
可在「运行记录管理」列表中查看、编辑或删除。

### 6. 暂停/继续
点击「暂停/继续」按钮，或设置画面长时间无变化后自动暂停（默认60s，建议20s以上）。

### 7. 调整参数
点击「设置」打开详细面板，保存后生效。

## 核心参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| SCALE_FACTOR | 0.5 | 截图缩放比例，降低计算量 |
| SSIM_THRESHOLD | 0.90 | 背景相似度阈值，用于局部变化过滤 |
| MIN_CHANGE_RATIO | 0.002 | 最小变化像素占比，低于此值判为无变化 |
| SIGNIFICANT_CHANGE_RATIO | 0.02 | 大面积变化直接通过阈值 |
| ALIGNED_CHANGE_THRESHOLD | 0.02 | 全局平移对齐后剩余变化面积上限，低于则视为纯平移运镜 |
| BASIC_CORR_THRESHOLD | 0.995 | 基础检测快速相似度阈值 |
| FULL_CORR_THRESHOLD | 0.998 | 完整检测快速相似度阈值 |
| FLOW_FEATURE_COUNT | 150 | 光流法提取的特征点数量 |
| HASH_THRESHOLD | 0 | 汉明距离阈值，≤此值视为重复帧（0 表示完全相同） |
| FILTER_TRIGGER_WINDOW_MS | 200 | 触发完整过滤的时间窗口（毫秒） |
| FILTER_TRIGGER_COUNT | 3 | 窗口内基础检测新张数达到此值触发完整过滤 |
| FULL_FILTER_HOLD_SEC | 5 | 触发后保持完整过滤的秒数 |
| JingzhiShiJian | 60 | 静止自动暂停秒数 |
| OPED_DETECTION_ENABLED | True | 启用 OP/ED 自动检测 |
| OPED_WINDOW_SEC | 90 | 哈希匹配窗口长度（秒） |
| OPED_HASH_MAX_DIST | 12 | 单个哈希汉明距离上限 |
| OPED_HASH_WIN_RATIO | 0.6 | 窗口内相似帧比例要求 |

## 工作原理

1. **屏幕捕获**：dxcam 以最高帧率抓取屏幕区域，缩放至指定尺寸。
2. **原始差分**：计算相邻帧绝对差，根据画面平均亮度自适应阈值二值化。
3. **基础检测**：
   - 若归一化相关系数极高且变化面积极小 → 判为静止/极慢平移
   - 若变化面积超过大面积阈值 → 直接判为新张。
   - 否则标记为可疑，进入动态触发统计。
4. **动态触发**：在时间窗口内，若基础检测的新张计数达到阈值，则启动完整过滤。
5. **完整检测**：
   - 全局平移对齐消除摄像机运镜；若对齐后变化极小 → 判为平移运镜。
   - 分析连通域 → 局部动作异常则过滤。
   - 光流方向一致性分析 → 同向运动判为多层运镜，过滤。
   - 光流径向一致性及位移-距离相关性 → 缩放运镜，过滤。
   - 剩余变化结合背景相似度判断是否为新作画。
6. **哈希去重**：利用感知哈希（16x16）消除短时重复帧，严格模式下汉明距离必须为0。
7. **OP/ED 检测**：
   - 每秒计算当前帧的 64 位 pHash 并存储。
   - 定期用滑动窗口搜索历史哈希序列，寻找与最近窗口汉明距离最小的区间。
   - 通过状态机连续确认匹配区间，归入模板或新建模板。
   - 当区间长度达标（≥20秒）后，通过 FFT 互相关精确定位起止时间，扣除该区间内的累计张数和时长。
8. **统计与显示**：计算每秒张数（扣除已识别OP/ED的时间与张数），更新波形，并分类统计各过滤原因的数量。
9. **运行记录**：将当前会话的详细统计数据 保存为 JSON 记录，方便后续查阅。

## 自定义界面

- **透明度**：可通过 ALPHA 参数调整窗口透明度（0.0~1.0）。
- **颜色主题**：在设置面板中启用“自定义调色板”，可为界面元素、波形线条等指定颜色。点击保存后生效。
- **布局记忆**：可保存多套面板排列，适应不同屏幕。

## 注意事项

- 程序默认捕获主显示器。若目标窗口在其他屏幕，需修改 dxcam 的显示器参数。
- 截图区域建议仅包含动画画面，排除弹幕、播放器 UI 等固定元素。
- 完整检测计算量大，程序已内置动态触发机制，仅在必要时启用。
- 关闭设置窗口或主窗口时，配置会自动保存至 `cel_counter_settings.json`。
- OP/ED 检测依赖于重复出现的画面序列，对仅播放一次的片段无法自动识别。若误识别，可通过重置统计清除。
- 运行记录保存在 `cel_counter_records.json`，手动编辑需谨慎，避免 JSON 格式错误。
