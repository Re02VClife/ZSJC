


# 动画作画张数计数器

【项目简介】
本工具用于实时检测并统计动画视频中的“作画张数”（cel），
即每秒出现的新画面数量。通过屏幕截图连续分析相邻帧的变化，
检测流程融合了自适应二值化变化检测、相位相关全局对齐、稀疏光流一致性判断、
平均哈希重复帧过滤以及动态触发式完整过滤机制，
最终实现实时区分真正作画张数与运镜/局部变化
自动排除摄像机运镜、图层平移等干扰，帮助你快速评估动画的作画密度和节奏。


这款轻量级工具能提供即时、直观的量化数据。直观展示作画经费、便于进行各个动画之间的赛博斗蛐蛐
仅限于2D动画、受算法局限性，检测出来的数据只是相对准确而不是绝对精准；
测出来高数据只代表原画、中割等制作人员付出的血汗较多并不意味着经费一定爆炸

本项目没有任何参考，出现各种奇怪的BUG属于正常现象
纯cpu计算，未使用CUDA，目前不知道兼不兼容Linux、mac

## 功能特性

- **屏幕实时捕获**：基于 `dxcam` 高速截取屏幕，可自由调整截图区域。
- **双重检测管线**：
  - 基础检测：快速模板匹配 + 差分，维持低负载运行。
  - 完整检测：背景对齐、光流图层分离、局部运动分析、SSIM 背景相似度。
    
- **多种过滤策略**：
  - 全局平移运镜过滤（对齐残差判断）
  - 多层平移/图层分离过滤（光流方向一致性）
  - 重复帧过滤（感知哈希，汉明距离）
  - 局部变化过滤（连通域面积、包围盒、宽高比分析）
  - 底部字幕智能屏蔽（基于对比度密度）
- **动态触发机制**：基础检测在短时间内检测到一定数量的新张后，自动开启完整过滤，平衡性能与准确度。
- **实时数据显示**：
  - 实时张数（张/秒）、总张数、运行时长、平均张数
  - 智能状态提示（静止、运镜、一拍一/二/三等）
- **双波形图**：
  - 波形1（60秒）：实时过滤速率 vs 原始变化速率
  - 波形2（25分钟）：平滑后的长期趋势
- **可调参数面板**：所有检测阈值、开关、颜色均可在设置窗口实时修改并保存。
- **自定义截图区域**：提供可视化矩形拖拽界面，精确选定检测区域。
- **实时预览**：支持稠密光流、稀疏光流、差分残影、运动轨迹叠加等多种可视化。
- **布局管理**：支持多套窗口布局的保存与加载。
- **自动暂停**：指定秒数无新张后自动暂停，避免资源浪费。
- **性能**： CPU9700x 功能全开占用10%，仅供参考

## 快速上手

### 1. 运行程序
```bash
python main.py
```
窗口将以透明无边框形式出现在屏幕左上角。鼠标尖尖对准文字，长按即可实现拖动

### 2. 设定截图区域
点击「设置」→「显示截图范围」，拖动红色矩形框选择要分析的画面区域，点击矩形外任意位置或在设置中关闭。

框的大小在设置面板中调整（截图区域比例）CROP_RATIO自动计算。

### 3. 开始检测
播放动画或任意视频（最好是2D动画），程序运行时会自动统计新作画张数。
第一次使用本程序时，在确保显示器的刷新率大于60hz，
在红框范围内做一些较猛的变化（如网页页面快速上下翻滚、拖动窗口猛甩、高帧率3D游戏转动视角等）
过程中看到左上角实时张数大于40（峰值48）即为设备性能合格

### 4. 查看结果
- 左上角信息面板：实时张数、总张数、平均张数、当前状态。
- 右侧两个波形图：短期与长期张数速率变化。
- 点击设置，查看预览图和更详细的过滤信息
### 5. 暂停/继续
点击「暂停/继续」按钮，或设置画面长时间无变化后自动暂停（默认60s，建议20s以上）。

### 6. 调整参数
点击「设置」打开详细面板，保存后生效。

## 核心参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| SCALE_FACTOR | 0.5 | 截图缩放比例，降低计算量 |
| SSIM_THRESHOLD | 0.90 | 背景相似度阈值，用于局部变化过滤 |
| MIN_CHANGE_RATIO | 0.002 | 最小变化像素占比，低于此值判为无变化 |
| SIGNIFICANT_CHANGE_RATIO | 0.01 | 大面积变化直接通过阈值 |
| ALIGNED_CHANGE_THRESHOLD | 0.012 | 全局平移对齐后剩余变化面积上限，低于则视为纯平移运镜 |
| FLOW_FEATURE_COUNT | 200 | 光流法提取的特征点数量 |
| HASH_THRESHOLD | 1 | 汉明距离阈值，≤此值视为重复帧 |
| FILTER_TRIGGER_COUNT | 3 | 触发完整过滤所需的基础检测新张数 |
| SUBTITLE_BOTTOM_RATIO | 0.12 | 底部屏蔽区域高度占比，设为 0 则不屏蔽 |
| JingzhiShiJian | 60 | 静止自动暂停秒数 |

完整列表见设置面板，由于参数不设限，乱填可能会引发不可名状的bug，调整时尽量小幅度调整，可随时恢复默认值。

## 工作原理

1. **屏幕捕获**：dxcam 以最高帧率抓取屏幕区域，缩放至指定尺寸。
2. **原始差分**：计算相邻帧绝对差，根据画面平均亮度自适应阈值二值化。
3. **基础检测**：
   - 若模板匹配相关度极高且变化面积极小 → 判为静止/极慢平移。
   - 若变化面积超过大面积阈值 → 直接判为新张。
   - 否则标记为可疑，进入动态触发统计。
4. **动态触发**：在时间窗口内，若基础检测的新张计数达到阈值，则启动完整过滤。
5. **完整检测**：
   - 背景对齐消除全局平移；若对齐后变化极小 → 判为平移运镜。
   - 分析连通域 → 局部动作异常则过滤。
   - 光流方向一致性分析 → 同向运动判为多层运镜，过滤。
   - 剩余变化结合 SSIM 判断是否为新作画。
6. **哈希去重**：利用感知哈希消除短时重复帧。
7. **统计与显示**：计算每秒张数，更新波形，并分类统计各过滤原因的数量。

## 自定义界面

- **透明度**：可通过 ALPHA 参数调整窗口透明度（0.0~1.0）。
- **颜色主题**：在设置面板中启用“自定义调色板”，可为界面元素、波形线条等指定颜色。点击保存后生效
- **布局记忆**：可保存多套面板排列，适应不同屏幕。

## 注意事项

- 程序默认捕获主显示器。若目标窗口在其他屏幕，需修改 dxcam 的显示器参数。
- 截图区域建议仅包含动画画面，排除弹幕、播放器 UI 等固定元素。
- 完整检测计算量大，程序已内置动态触发机制，仅在必要时启用。
- 关闭设置窗口或主窗口时，配置会自动保存至 `cel_counter_settings.json`。
